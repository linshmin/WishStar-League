# WishStar League 團隊網站正式版架構規劃

## 1. 專案目標
建立一個只對團隊夥伴開放的網站，提供：
- 夥伴註冊與管理員審核啟用
- 管理員與團隊領導可發佈公告、工作要點、銷售技巧、商品訊息、教育訓練
- 夥伴登入後可查詢上述內容
- 每位夥伴可填寫自己的工作紀錄與留言
- 團隊領導可查看自己團隊夥伴的工作紀錄與留言，並給予回饋
- 管理員可查看全部夥伴資料、全部工作紀錄、全部文章內容，並進行管理
- 文章系統五分類標籤篩選（公告/工作要點/銷售技巧/商品訊息/教育訓練）
- 搜尋功能可依關鍵字查看文章

## 2. 角色與權限

### 管理員 SHMIN
- 預設帳號：`admin` / 密碼：`admin123`
- 審核夥伴註冊申請、啟用或刪除帳號
- 可新增新的團隊名稱
- 查看所有夥伴資料（先選擇團隊名稱，再顯示該團隊人員名單）
- 查看全部工作紀錄與留言、給予回饋
- 發佈與管理所有文章與公告
- 管理註冊夥伴之分類、團隊與權限、可協助設定登入密碼
- 可開啟團隊領導人的權限

### Mika 領導人
- 預設帳號：`mika` / 密碼：`mika0812`
- 查看所有團隊夥伴資料（先選擇團隊名稱，再顯示該團隊人員名單）
- 查看所有團隊夥伴的工作紀錄與留言、給予回饋
- 發佈文章與公告給自己的團隊或指定團隊或所有人

### 團隊領導
- 預設帳號範例：`leader1` / 密碼：`leader123`
- 查看所屬團隊夥伴資料
- 查看所屬團隊夥伴的工作紀錄與留言、給予回饋
- 發佈文章與公告給所屬團隊或所有人

### 一般夥伴
- 自行註冊帳號與密碼，Email 可不填
- 註冊後狀態為 `pending`，管理員審核通過後改為 `approved`
- 登入後查看公告、工作要點、銷售技巧、商品訊息、教育訓練
- 填寫自己的工作紀錄、查看收到的回饋
- 忘記密碼可透過 Email 找回

## 3. 註冊與審核流程
1. 夥伴進入註冊頁，填寫姓名、帳號、手機、所屬團隊（下拉選單 8 個團隊）、密碼
2. Email 選填
3. 系統建立帳號，狀態為 `pending`
4. 管理員後台待審核列表出現此申請
5. 管理員確認後將帳號改為 `approved`
6. 帳號啟用後即可登入

## 4. 登入後首頁規劃
- 五大分類快速入口（公告/工作要點/銷售技巧/商品訊息/教育訓練）
- 最新文章列表
- 自己的工作紀錄摘要
- 主管回饋醒目提醒

## 5. 文章系統
- 五分類標籤篩選
- 關鍵字搜尋
- 管理員與團隊領導可發文、指定發佈對象（全體/特定團隊）

## 6. 工作紀錄系統
- 一般夥伴可每日填寫工作紀錄與留言
- 團隊領導可查看所屬團隊紀錄並給予回饋
- Mika 領導人可查看所有團隊紀錄
- 管理員可查看全部紀錄並給予回饋

## 7. 技術架構（前端原型）
- **index.html**：頁面結構（SPA 架構，所有頁面在同一 HTML 中切換）
- **styles.css**：星光主題視覺設計，響應式佈局
- **app.js**：前端互動邏輯，localStorage 模擬資料持久化
- **SYSTEM_PLAN.md**：本文件

## 8. 預設團隊
文玲團隊、玫眞團隊、鳳思團隊、鳳茹團隊、喬卉團隊、碧雲團隊、珮慈團隊、俋準團隊

## 9. 正式版建議
- 後端：Node.js + Express 或 Python + FastAPI
- 資料庫：PostgreSQL 或 MongoDB
- 認證：JWT Token + bcrypt 密碼加密
- 部署：Docker 容器化，Nginx 反向代理
- 安全性：HTTPS、CSRF 防護、Rate Limiting
